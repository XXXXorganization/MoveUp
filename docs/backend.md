# Moveup 后端服务文档

## 📋 项目概述
Moveup 后端服务为移动端运动跑步软件提供数据支撑，负责用户管理、运动数据追踪、社交互动等核心功能。采用 Node.js + Express 框架，配合 PostgreSQL 和时序数据库构建。

## 🎯 核心功能模块

### 1. 用户管理模块
- **用户注册与登录**
  - 手机号验证码登录
  - JWT token 身份认证
- **个人资料管理**
  - 头像上传与裁剪
  - 个人信息编辑（昵称、性别、生日、身高体重）
  - 运动偏好设置（跑步时间、距离目标）
- **账号安全**
  - 密码找回/修改
  - 设备管理（登录设备记录）
  - 账号注销功能

### 2. 运动数据模块
- **实时运动追踪**
  - GPS 轨迹记录与纠偏
  - 配速、距离、时长实时计算
  - 卡路里消耗算法（基于心率、体重、配速）
  - 心率数据接入（支持蓝牙设备）
- **运动记录管理**
  - 历史运动数据存储
  - 运动详情查看（轨迹回放、分段配速）
  - 数据统计分析（周/月/年报表）
- **路线管理**
  - 常用路线保存
  - 热门路线推荐
  - 路线分享与导入

### 3. 智能指导模块
- **个性化计划**
  - AI 生成训练计划（基于用户水平和目标）
  - 每日训练任务推送
  - 训练完成度跟踪
- **语音指导**
  - 实时语音播报（配速、距离提醒）
  - 分段教练建议（根据心率调整配速）
- **健康建议**
  - 跑后拉伸指导
  - 伤病预防提醒

### 4. 社交互动模块
- **好友系统**
  - 好友添加/删除
  - 好友搜索（手机号、昵称）
  - 好友运动动态查看
- **社区内容**
  - 动态发布（文字、图片、运动记录）
  - 点赞、评论、转发
  - 话题标签（#晨跑打卡 #备赛训练）
- **排行榜**
  - 好友榜（周跑量、月跑量）
  - 同城榜
  - 全球挑战赛

### 5. 挑战与激励模块
- **任务系统**
  - 日常任务（连续打卡、完成目标）
  - 成就徽章（首马完成、累计1000公里）
- **实景挑战**
  - 虚拟路线挑战（城市地标打卡）
  - 团队接力赛
- **会员服务**
  - 付费会员权益（专属计划、高级数据）
  - 积分商城兑换

## 🔧 技术架构

### 服务分层
```
┌─────────────────┐
│   API网关层     │ 负载均衡、限流、鉴权
├─────────────────┤
│   业务服务层    │ 用户服务、运动服务、社交服务
├─────────────────┤
│   数据服务层    │ 关系型DB、时序DB、缓存
├─────────────────┤
│   基础设施层    │ 对象存储、消息队列、CDN
└─────────────────┘
```

### 关键服务设计
- **微服务拆分**：按业务域拆分为用户服务、运动服务、社交服务
- **消息队列**：RabbitMQ 处理异步任务（图片处理、数据计算）
- **定时任务**：每日数据汇总、排行榜更新
- **实时通信**：WebSocket 服务推送实时运动数据

## 📊 数据存储方案
- **PostgreSQL**：用户信息、社交关系、运动记录元数据
- **InfluxDB**：实时心率、GPS 点序列、分段配速
- **Redis**：Session 管理、实时排行榜、热门缓存
- **MinIO/OSS**：用户头像、运动截图、社区图片

## 🔒 安全策略
- **数据传输**：全站 HTTPS，敏感字段加密存储
- **权限控制**：基于 RBAC 的接口权限
- **防攻击**：API 限流、SQL 注入过滤、XSS 防护
- **数据备份**：每日自动备份，7天增量备份

## 📈 性能指标
- **并发支持**：单服务节点 1000+ QPS
- **响应时间**：95% 接口 < 200ms
- **可用性**：99.9% 服务可用性
