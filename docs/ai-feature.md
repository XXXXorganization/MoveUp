# AI 功能说明文档

本项目在 **MoveUp** 跑步应用中集成了一个基于大语言模型（LLM）的“智能跑步教练”系统。该功能不局限于简单的聊天，而是实现了一个能够感知用户数据并直接执行任务的 **AI Agent（智能体）**。

## 1. 使用模型
- **模型名称**: `Qwen/Qwen2.5-7B-Instruct` (通义千问 2.5 开源版) [cite: 3.3]
- **API 平台**: 硅基流动 (SiliconFlow) [cite: 1.3]

## 2. 实现功能

### 📊 私有数据驱动的智能分析
AI 能够“读取”后端数据库中存储的该用户真实的跑步历史记录（包括总里程、总时长、平均配速等）以及当前的每周计划表。
- **功能描述**: 当用户询问“我的运动情况如何”时，AI 会基于真实数据给出分析，而不是空泛的鼓励 [cite: 3.1]。
- **实现原理**: 后端在发起 AI 请求前，会先查询 `userRunsDB` 和 `planDataDB`，将数据脱敏后注入系统提示词（System Prompt）中。

### 📅 对话式计划生成 (AI Agent 自动执行)
这是本项目的核心亮点。AI 不仅能提供建议，还能直接修改用户的跑步计划表。
- **功能描述**: 用户通过自然语言下达指令（例如：“帮我周一早上6点排一个3公里的跑量”），AI 会在回复的同时生成一段隐藏指令 [cite: 3.1]。
- **自动执行逻辑**: 
    1. AI 输出符合协议的 JSON 字符串：`###PLAN:[{"day":"...", ...}]`。
    2. 后端逻辑利用 **正则表达式** 精准拦截并提取该指令 [cite: 3.1]。
    3. 后端自动解析 JSON 数组并将其写入 `planDataDB` 数据库 [cite: 3.1]。
    4. 用户退出聊天界面进入“计划”页面，即可看到 AI 自动添加的日程。

### 🛡️ 安全与稳定性
- **密钥安全**: API Key 严格存储在后端代码中（`index.js`），不向前端暴露，符合作业要求的安全性规范 [cite: 2.1]。
- **异常处理**: 针对大模型输出格式不稳定的问题，后端增加了正则表达式容错处理，确保即使 AI 在 JSON 后输出废话也能正确提取数据 [cite: 3.1]。

## 3. 技术架构简图
[前端 AItalk (发送文本)] -> [后端 Node.js (注入私有数据)] -> [云端 LLM (决策/生成指令)] -> [后端拦截器 (解析 JSON 并写入数据库)] -> [前端 Plan 页面 (同步显示)]
